1. MSI 개요
MSI(Mainstreet Investment Intelligence Brief)는 메인스트리트인베스트먼트 및 메인스트리트벤처스 투자담당임원을 위한 주간 투자 인텔리전스 브리프 시스템이다.
구분내용
하는 일 매주 월요일 기준 국내외 투자 유망 기업 5~7개를 조사하고 분석
재무, 리스크, 포트폴리오 적합성을 평가해 IC 메모로 합성
A4 한국어 HTML 문서로 출력 (이메일 발송 가능)
분석 결과를 기업별 마크다운 파일과 회의록으로 아카이빙
투자 포커스 Series A 이상 성장단계 (Seed, Pre-seed 제외)
M&A, Secondary, Pre-IPO
Deeptech/AI · K-content/IP · 브랜드 글로벌라이제이션 · Agtech/Foodtech · 디지털 헬스케어/시니어케어 · RWA/Fintech
2. 왜 이 구조가 필요한가
단일 AI 에이전트로 전체 브리프를 생성하면 분석의 편향과 검증 부재가 문제가 된다. MSI는 역할이 분리된 8개 에이전트를 순서대로 실행함으로써 각 분석 단계를 독립적으로 수행하고 다음 에이전트에 전달하는 파이프라인 구조를 택했다.
매주 실행 결과를 파일 시스템에 누적하여 시간이 지날수록 판단 품질이 개선되는 구조를 목표로 한다.
(watchlist.md, scoring_history.csv, rejected.md 참조)

6. 데이터 레이어 (루트 파일)
msi2가 매회 브리프 시작 시 순서대로 읽는 파일들. 각 파일은 특정 유형의 판단 실수 또는 기억 손실을 방지하기 위해 존재한다.
파일방지하는 것내용
thesis.md 기준 없는 판단 투자 철학 전문. "투자하지 않는 기업" 기준 포함. 모든 추천의 최우선 검증 기준.
watchlist.md 후보 기업 맥락 소실 장기 추적 후보 목록. status, next_trigger, concerns 포함.
rejected.md 동일 실패 반복 패스 기업 전체 기록. reconsideration_trigger 미달성 시 재추천 차단.
portfolio.csv 섹터 중복 투자 / 유동성 충돌 현재 투자 포지션 전체. 섹터 쏠림·follow_on 여부·엑싯 타이밍 충돌 방지.
scoring_history.csv rerating 신호 누락 기업별 IC 점수 이력. 주차별 score_delta로 RERATING·DETERIORATION 추적.
sector_map.yaml exposure_cap 초과 섹터 taxonomy 및 섹터별 투자 한도. portfolio2가 cap 초과 여부 계산에 사용.
peer_map.json 밸류에이션 기준 없는 판단 국내(KOSPI/KOSDAQ) + 글로벌(NASDAQ/NYSE) peer 배수 기준.
dd_questions.md 섹터별 핵심 질문 누락 섹터별 DD 체크리스트. finance2·risk2가 공개 정보로 답변 가능 여부를 확인.
7. Scoring 구조
IC Scorecard: 기업별 6개 항목 각 1~5점, 총 30점 만점
Market Opportunity
TAM 크기, 성장성, 시장 지배력 가능성
Growth
매출 성장률, 고객 증가세, 시장 침투율
Capital Efficiency
번 멀티플, ARR per $ Raised, 그로스 마진
Exit Visibility
M&A 가능성, IPO readiness, Secondary 유동성
Strategic Fit
메인스트리트 포트폴리오 시너지, 섹터 전략 부합도
Risk-Reward Ratio
리스크 대비 기대 수익, 밸류에이션 적정성

8. Memory 및 Provenance 규칙
절대 금지:
에이전트가 실행되지 않았는데 출력이 있는 것처럼 기록
에이전트 간 deliberation, 의견 충돌, conviction shift를 사후 합성
미결 항목을 해결된 것처럼 표기
에이전트 실패 시 기록 방식:
[Agent Timeout] [No Output Generated] [Execution Failed] [Incomplete Analysis]
log2 authenticity_note 필드: "This log summarizes generated outputs. It does not invent agent deliberation." — 매 회의록 frontmatter에 명시.

시스템의 목적은 폴리시된 산출물이 아니라 신뢰할 수 있는 시간적 의사결정 기록이다. Memory integrity is more important than presentation completeness.
9. Temporal Vocabulary
MSI 시스템 전반에서 사용되는 용어의 정확한 의미. 유사해 보이는 개념 간 구분이 중요하다.
RERATING
기준: score_delta ≥ +3
기존 판단의 구조적 변화. 기업 펀더멘털, 엑싯 가시성, 시장 포지션 등에서 실질적 변화가 있었음을 나타냄.
MOMENTUM과 구분: 단순 수치 개선이 아니라 판단 프레임 자체가 바뀐 경우.
MOMENTUM
기준: score_delta +1 또는 +2
긍정적 방향의 단기 시그널. 구조 변화까지는 아님. Key Signals에 반영하되 Strong Buy 상향의 근거로는 단독 사용 불가.
STABLE
기준: score_delta 0
conviction 유지 상태. 변화 없음이 아니라 변화가 판단에 영향을 주지 않는 수준임을 의미.
주의: STABLE이어도 Open Question이 미해결 상태일 수 있음. meeting-log의 Unresolved Items 확인 필요.
DETERIORATION
기준: score_delta ≤ −1
기존 판단 대비 악화. 해당 기업의 Conviction Shift 섹션에 명시 필수.
Carry-over
해결되지 않은 Open Question이 다음 브리프 사이클로 넘어가는 상태. 기록 위치: meeting-log.md의 Unresolved Items 섹션.
목적: 질문이 대화 컨텍스트 소실과 함께 사라지지 않도록 파일에 명시적으로 보존.
Rejection Memory
Pass 판정 이후 동일 기업이 재추천되는 것을 방지하고, 재검토 조건을 추적하는 구조.
단계동작
1memo2가 Pass 판정 시 rejected.md에 기업 정보 append
2reconsideration_trigger 필드에 재검토 조건 명시
3msi2가 신규 발굴 시 rejected.md 먼저 확인 → trigger 달성 여부 판단
4미달성 시: 해당 회차 브리프에서 조용히 제외 (재추천 없음)
5달성 시: Key Signals에 "재검토 트리거 달성" 명시 후 재평가
10. 현재 한계 및 실험 중인 부분
