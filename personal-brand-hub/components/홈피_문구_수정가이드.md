## 2. ⚠️ 고치고 "푸시"만 하면 라이브는 안 바뀜
라이브 서버는 git이 아니라 `/opt/swarm56/app` 의 **빌드된 복사본**을 `npm start` 로 돌린다.
- repo 편집 + 푸시 = GitHub 만 갱신 (사이트 그대로)
- **라이브 반영하려면 서버에서 새 코드 빌드 + 재시작 필요**

## 3. 라이브 반영 방법 (택1)
- **(A) Claude에게 요청**: "hero 제목을 ~로 바꿔줘" → Claude가 서버에서 수정 후 빌드·재시작 (1~2분).
- **(B) 직접 편집 후 재배포**: repo에서 문구 편집 → 푸시 → 서버에서 아래 실행
  ```bash
  ssh -i "<키>" ubuntu@54.116.19.34
  cd /opt/swarm56/v5build && git pull
  cd personal-brand-hub && npm ci && npm run build
  # 빌드 결과를 라이브로 스왑 후 재시작 (덮어쓰기 주의 — Claude/재배포 스크립트 권장)
  sudo systemctl restart swarm56-web
  ```
- **(C) 재배포 스크립트**(`deploy/`에 만들어두면 한 명령으로): 아직 미작성. 필요 시 Claude에게 요청.

## 참고
- 서버: AWS Lightsail `54.116.19.34` (ubuntu), 앱 `/opt/swarm56/app`, systemd `swarm56-web`.
- SSH 키: repo 밖 보관 권장.
